# Software Captain ခေတ်ကို ကြိုဆိုပါတယ် 🏴‍☠️

> ကျွန်တော့်အဖေအတွက်၊ ပင်လယ်ထဲမှာရော ဘဝမှာပါ အကြီးမြတ်ဆုံး သင်္ဘောသားတစ်ယောက်

## သင့်လမ်းကြောင်းကို ရေးဆွဲပါ

ကြီးမြတ်တဲ့ developer တစ်ယောက်ဖြစ်ဖို့ သင်ယူရမယ့်အရာတစ်ခုက လမ်းကြောင်းရှာတတ်ဖို့ပါ။

သင်နှစ်သက်တဲ့ project တွေကို ကြည့်ပါ။ idea ကနေ deployment အထိ ချောမွေ့စွာ သွားတဲ့ project တွေ။ စိန်ခေါ်မှုတွေကို ရင်ဆိုင်ပြီး ပိုမိုအားကောင်းလာတဲ့ project တွေ။ အဲဒီကနေ စပါ။ သူတို့ရဲ့ လမ်းကြောင်းရှာနည်းတွေကို လေ့လာပါ။ သူတို့ရဲ့ ခေါင်းဆောင်မှုနည်းလမ်းတွေကို သင်ယူပါ။ သူတို့ရဲ့ team ကို ဘယ်လို စိတ်အားထက်သန်စေပြီး လမ်းကြောင်းပေါ်မှာ ထိန်းထားတယ်ဆိုတာ နားလည်ပါ။

ကျွန်တော်သိတဲ့ အကောင်းဆုံး developer တွေက အကောင်းဆုံး လမ်းကြောင်းရှာသူတွေပါ။ သူတို့မှာ အောင်မြင်တဲ့ project တွေရဲ့ မြေပုံတွေ၊ best practice တွေကို ညွှန်ပြတဲ့ ကွန်ပါစ်တွေ၊ လေ့လာမှုတွေနဲ့ ပြည့်နေတဲ့ log တွေ ရှိကြတယ်။

ကျွန်တော်က ဒါကို ကျွန်တော့်အဖေ ပင်လယ်အစစ်ကို လမ်းကြောင်းရှာတာကို ကြားရာကနေ သင်ယူခဲ့တယ်။ သူက မတူညီတဲ့ နိုင်ငံတွေက crew တွေနဲ့ ညှိနှိုင်းတာ၊ မတူညီတဲ့ ဘာသာစကားတွေ ပြောတဲ့ officer တွေကို စီမံတာ၊ နိုင်ငံတကာ ရေပြင်တွေကို ဖြတ်သန်းရင်း timezone ပြောင်းလဲမှုတွေရဲ့ စိန်ခေါ်မှုတွေကို ရင်ဆိုင်တာတွေရဲ့ ပုံပြင်တွေနဲ့ အိမ်ပြန်လာတတ်တယ်။ Remote development team တွေကို ဦးဆောင်တာနဲ့ နှိုင်းယှဉ်ကြည့်ရင် ထူးဆန်းတဲ့ နှိုင်းယှဉ်မှုတွေ ရှိတယ် - လူတွေကို မျက်နှာချင်းဆိုင် မတွေ့ဖူးတဲ့ လူတွေနဲ့ ညှိနှိုင်းနေရတာ၊ မတူညီတဲ့ ဆက်သွယ်ရေးပုံစံတွေကို စီမံနေရတာ၊ အဆုံးမရှိတဲ့ timezone တွေကို ဖြတ်သန်းရင်း အလုပ်လုပ်နေရတာ။

ဒါက coding ပဲ မဟုတ်ပါဘူး။ ဒါက ခေါင်းဆောင်မှု ဘယ်လို အလုပ်လုပ်တယ်ဆိုတာပါ။ 🧭

## ဆိပ်ကမ်းတိုင်းကို မသိမချင်း မစောင့်ပါနဲ့

ကြီးမြတ်တဲ့ captain တစ်ယောက်ဖြစ်ဖို့ သင်ယူရမယ့်အရာတစ်ခုက လမ်းကြောင်းရှာတတ်ဖို့ပါ။

ဒါက နူးညံ့တဲ့ အမှန်တရားတစ်ခုပါ - သင်က နည်းပညာတိုင်းကို သိဖို့ မလိုပါဘူး။ သင်က လမ်းကြောင်းရှာတတ်ဖို့ပဲ လိုပါတယ်။

ကျွန်တော်သိတဲ့ အကောင်းဆုံး captain တွေက project တစ်ခု၊ team တစ်ခု၊ သင်္ဘောငယ်တစ်စီးနဲ့ စခဲ့ကြတယ်။ သူတို့က လမ်းကြောင်းရှာတာကို တကယ်လမ်းကြောင်းရှာရင်း သင်ယူခဲ့ကြတယ်။ သူတို့က ဦးဆောင်တာကို တကယ်ဦးဆောင်ရင်း သင်ယူခဲ့ကြတယ်။ သူတို့က မုန်တိုင်းတွေကို ရင်ဆိုင်တာကို တကယ်မုန်တိုင်းတွေကို ရင်ဆိုင်ရင်း သင်ယူခဲ့ကြတယ်။

သင်ရှိတဲ့အရာနဲ့ စပါ။ သင်ရှိတဲ့နေရာနဲ့ စပါ။ သင်ရှိတဲ့ crew နဲ့ စပါ။ ပင်လယ်တွေက ကျန်တာတွေကို သင်ပေးမှာပါ။ ⚓

## သင်ပါဝင်ချင်တဲ့ Team ကို ဦးဆောင်ပါ

သင်ဘယ်လို ခေါင်းဆောင်တစ်ယောက် ဖြစ်ချင်တယ်ဆိုတာ ရှာဖွေဖို့ အကောင်းဆုံးနည်းလမ်းက ဦးဆောင်တာကို စရုံပါပဲ။

"ငါ ဘာတွေ build လုပ်သင့်လဲ?" လို့ မေးမယ့်အစား "ငါ ဘယ်လို team မျိုးနဲ့ ပါဝင်ချင်လဲ?" လို့ မေးကြည့်ပါ။

သင်ပါဝင်ခဲ့တဲ့ team တွေကို ကြည့်ပါ။ နည်းနည်း စိတ်အနှောင့်အယှက် ဖြစ်စေတဲ့ team တွေ။ "အတူတကွ အလုပ်လုပ်ဖို့ ပိုကောင်းတဲ့ နည်းလမ်း ရှိရမယ်" လို့ စဉ်းစားစေတဲ့ team တွေ။ ဒါက မကြာခဏ သင့်ရဲ့ အကောင်းဆုံး စတင်တဲ့နေရာ ဖြစ်တယ်။

ကျွန်တော်သိတဲ့ developer တွေက နှစ်ပေါင်းများစွာ management theory တိုင်း၊ team structure တိုင်း၊ communication pattern တိုင်းကို လေ့လာကြတယ်။ သူတို့က ဘာမှ ဦးဆောင်မှု မလုပ်ကြဘူး ဘာလို့လဲဆိုတော့ သူတို့က ပြီးပြည့်စုံတဲ့ ခေါင်းဆောင်မှု အခွင့်အလမ်း လာမယ့်ကို စောင့်နေကြတယ်။

အခြား developer တွေကတော့ မပြီးပြည့်စုံတဲ့၊ တစ်ခါတစ်ရံ ရှုပ်ထွေးတဲ့ team တွေကို ဦးဆောင်နေကြတယ်။ သူတို့က သင်ယူနေကြတယ်။ သူတို့က ပိုကောင်းလာနေကြတယ်။ နောက်ဆုံးမှာ သူတို့က အံ့သြစရာ team တွေကို build လုပ်နေကြတယ်။

သင်သိတဲ့အရာနဲ့ စပါ။ သင်ကြုံတွေ့ခဲ့တဲ့အရာနဲ့ စပါ။ team dynamics အကြောင်း နည်းနည်း စိတ်အနှောင့်အယှက် ဖြစ်စေတဲ့အရာနဲ့ စပါ။

ကျန်တာတွေက သဘာဝအတိုင်း လိုက်လာမှာပါ။ 👥

## Cadet ကနေ Captain အထိ - အတွေ့အကြုံရဲ့ ခရီးစဉ် 🚢

Captain ရဲ့ ရိုမန်တစ်ပုံစံမှာ ပျောက်ဆုံးနေတဲ့အရာတစ်ခုက - captain တိုင်းက တစ်ခါက cadet ဖြစ်ခဲ့တယ်။ Software Captain တိုင်းက တစ်နေရာရာမှာ စခဲ့ကြတယ်၊ သူတို့ရဲ့ ကျွမ်းကျင်မှုရဲ့ အခြေခံကို တည်ဆောက်တဲ့ ခက်ခဲတဲ့၊ မကြာခဏ ကျေးဇူးမတင်ရတဲ့ အလုပ်တွေကို လုပ်ခဲ့ကြတယ်။ 🎓⚓

Software Captain ဖြစ်လိုတဲ့ လူတိုင်းက cadet ဖြစ်ပြီး စကြတယ် - သူတို့ရဲ့ ပထမဆုံး "Hello World" ကို ရေးတာ၊ သူတို့ရဲ့ ပထမဆုံး infinite loop ကို debug လုပ်တာ၊ သူတို့ရဲ့ machine မှာ အလုပ်လုပ်တဲ့ code က production မှာ ထူးဆန်းစွာ ပျက်သွားတာကို သင်ယူတာ။

အစောပိုင်း လေ့ကျင့်မှုပြီးနောက်၊ Software Captain တွေက "လက်ဖြင့် အလုပ်လုပ်တဲ့" အခန်းကဏ္ဍတွေကို ဖြတ်သန်းကြတယ် - legacy code တွေကို ရှင်းလင်းတာ၊ ထူးဆန်းတဲ့ production issue တွေကို debug လုပ်တာ၊ test တွေ ရေးတာ၊ documentation ရေးတာ။ သင်္ဘောရဲ့ hull ကို rust ဖယ်ရှားပြီး ပြန်ဆေးတာလိုပဲ၊ ဒီအလုပ်က မကြာခဏ user တွေအတွက် မမြင်ရပေမယ့် သင်္ဘောရဲ့ ရှည်လျားမှုအတွက် အရေးကြီးတယ်။ ဒီလက်တွေ့ အတွေ့အကြုံ နှစ်ပေါင်းများစွာ စုဆောင်းပြီးနောက်မှသာ တစ်ယောက်ယောက်က Software Captain ဖြစ်ဖို့ အရည်အချင်း ရှိလာတယ်။

ဒါကြောင့် သင်က သင့်ရဲ့ cadet နှစ်တွေမှာ ရှိနေသေးရင်၊ ခရီးစဉ်ကို မမြန်ဆန်ပါနဲ့။ ခက်ခဲတဲ့ အလုပ်ကို ချစ်တတ်အောင် သင်ယူပါ။ အခြေခံတွေကို ကျွမ်းကျင်အောင် လုပ်ပါ။ အတွေ့အကြုံကို အောက်ခြေကနေ တည်ဆောက်ပါ။ ဘာလို့လဲဆိုတော့ သင်က နောက်ဆုံးမှာ သင့်ရဲ့ captain ရဲ့ အနားကြောင်းတွေကို ရရှိမယ့်အခါ၊ သင်က သူတို့ကို ရည်မှန်းချက်ကြောင့် မဟုတ်ဘဲ အတွေ့အကြုံကြောင့် ရရှိခဲ့တယ်ဆိုတာ သိမှာပါ။ 🏆

## Digital Seas တွေကို လမ်းကြောင်းရှာခြင်း 🌊

Software Captain တိုင်းက သူတို့ရဲ့ ခရီးစဉ်မှာ မုန်တိုင်းတွေ၊ ရေခဲတောင်တွေ၊ ပင်လယ်ဓားပြတွေကို ရင်ဆိုင်ရမှာပါ။ အကောင်းဆုံး captain တွေက ဒီအန္တရာယ်ရှိတဲ့ ရေပြင်တွေကို ဘယ်လို လမ်းကြောင်းရှာကြသလဲ - ⛈️⚓ Captain တိုင်းက technical debt ရဲ့ မုန်တိုင်းကို ရင်ဆိုင်ရတယ် - သင့်သင်္ဘောကို hull ပေါ်က barnacle တွေလို နှေးကွေးစေတဲ့ စုဆောင်းထားတဲ့ code။ Legacy system တွေက မျက်နှာပြင်အောက်မှာ ပုန်းကွယ်နေတဲ့ ရေခဲတောင်တွေလို၊ သင့်ရဲ့ ဂရုတစိုက် စီစဉ်ထားတဲ့ feature တွေကို နစ်မြုပ်စေဖို့ စောင့်နေတယ်။

🧊 Digital seas တွေမှာ၊ ပင်လယ်ဓားပြတွေက ပုံစံမျိုးစုံနဲ့ လာကြတယ် - ဆင်တူတဲ့ feature တွေ လွှတ်တဲ့ ပြိုင်ဘက်တွေ၊ ပိုမြန်အောင် ship လုပ်ဖို့ market pressure၊ သင့်ရဲ့ crew တစ်ခုလုံးကို အစားထိုးမယ်လို့ ကတိပေးတဲ့ AI feature တွေ။ ဒီပင်လယ်ဓားပြတွေက သင့်ရဲ့ cargo (features) ကို ခိုးယူပြီး သင့်ရဲ့ သင်္ဘော (product) ကို နစ်မြုပ်စေချင်ကြတယ်။ 🤖🏴‍☠️ သင့်ရဲ့ crew က အရာအားလုံးပါ။ Captain တစ်ယောက်က သူ့ရဲ့ crew မရှိရင် ဘာမှ မဟုတ်ပါဘူး - rigging (frontend) ကို ကိုင်တွယ်နိုင်တဲ့၊ engine room (backend) ကို လမ်းကြောင်းရှာနိုင်တဲ့၊ သင်္ဘောကို ချောမွေ့စွာ လည်ပတ်စေနိုင်တဲ့ (DevOps) ကျွမ်းကျင်တဲ့ crew တွေ။

မုန်တိုင်းတွေ ပြင်းထန်နေပြီး ပင်လယ်ဓားပြတွေ ပိတ်မိနေတဲ့အခါ၊ သင့်ရဲ့ ကွန်ပါစ်က သင့်ကို လမ်းကြောင်းပေါ်မှာ ထိန်းထားတဲ့အရာပါ။ Software development မှာ၊ ဒါက သင့်ရဲ့ team ရဲ့ တန်ဖိုးတွေ၊ သင့်ရဲ့ coding principles တွေ၊ သင့်ရဲ့ quality ကို ကတိကဝတ်ပြုမှုပါ။ အကောင်းဆုံး captain တွေမှာ အားကောင်းတဲ့ ကိုယ်ကျင့်တရား ကွန်ပါစ် ရှိကြတယ်။ သူတို့က ပိုမြန်အောင် ship လုပ်ဖို့ security ကို စွန့်လွှတ်မှု မလုပ်ကြဘူး။ သူတို့က နောက်ပိုင်းမှာ သင်္ဘောကို နစ်မြုပ်စေနိုင်တဲ့ ထောင့်တွေကို မဖြတ်ကြဘူး။ သူတို့က သူတို့ရဲ့ crew နဲ့ cargo ရဲ့ ရေရှည် ကျန်းမာရေးအတွက် မှန်တဲ့အရာကို အခြေခံပြီး ဆုံးဖြတ်ချက်တွေ ချကြတယ်။ ⚓

## Tortuga ကို ရှာဖွေခြင်း - Captain ရဲ့ Haven 🏴‍☠️

Software Captain တိုင်းက Tortuga လိုအပ်တယ် - captain တွေ အနားယူ၊ လောင်စာဖြည့်၊ သူတို့ရဲ့ စစ်ပွဲပုံပြင်တွေ မျှဝေဖို့ စုဝေးတဲ့နေရာ။ Digital seas တွေမှာ၊ Tortuga က ရုပ်ပိုင်းဆိုင်ရာ ကျွန်းတစ်ကျွန်း မဟုတ်ပါဘူး၊ ဒါပေမယ့် command ရဲ့ ဝန်ထုပ်ကို နားလည်တဲ့ လက်တွဲဖော် captain တွေရဲ့ community တစ်ခုပါ။ 🏴‍☠️🍺 Tortuga က ပုံစံမျိုးစုံနဲ့ ရှိတယ် - captain တွေက သူတို့ရဲ့ သင်္ဘောတွေကို showcase လုပ်ပြီး code treasure တွေ မျှဝေတဲ့ GitHub၊ captain တွေက သူတို့ရဲ့ ခရီးစဉ်တွေနဲ့ bug တွေနဲ့ စစ်ပွဲတွေကို မျှဝေတဲ့ Dev.to၊ captain တွေက သူတို့ရဲ့ သင်္ဘော ရေတွေ စုပ်ယူနေတဲ့အခါ သွားတဲ့ Stack Overflow၊ captain တွေ သေးငယ်တဲ့ group တွေမှာ စုဝေးတဲ့ Discord/Slack community တွေ၊ digital seas တွေရဲ့ နေရာအားလုံးကနေ captain တွေ စုဝေးပြီး ရှာဖွေတွေ့ရှိမှုတွေ မျှဝေတဲ့ conference တွေ။ Tortuga ကို ဝင်တဲ့ Software Captain တိုင်းက မရေးထားတဲ့ code တစ်ခုကို လိုက်နာတယ် - သင့်ရဲ့ knowledge ကို မျှဝေပါ၊ ပင်လယ်ဓားပြတွေကို ကူညီပါ၊ သင့်ရဲ့ စစ်ပွဲပုံပြင်တွေကို ပြောပါ၊ သင့်ရဲ့ crew ကို လေးစားပါ၊ နောက်တစ်ပတ်ကို သင့်ရဲ့ expertise ကို မျှဝေခြင်းဖြင့် ဝယ်ပါ။

## Captain ရဲ့ စစ်မှန်တဲ့ ကွန်ပါစ် 🧭

အခု၊ captain တွေအများစု မသင်ယူဖူးတဲ့အရာတစ်ခုက - သင်ရှိတဲ့နေရာကို သိခြင်းနဲ့ သင်သွားချင်တဲ့နေရာကို သိခြင်း ကြားမှာ ကွာခြားချက် ရှိတယ်။ ✨🗺️

သင်တွေ့ရမယ်၊ captain တိုင်းမှာ ကွန်ပါစ် တစ်ခု ရှိတယ်။ အများစုက ဒါက မြောက်ဘက်ကို ညွှန်တယ်လို့ ထင်ကြတယ်၊ ဒါပေမယ့် စစ်မှန်တဲ့ ကွန်ပါစ်တွေ ဘယ်လို အလုပ်လုပ်တယ်ဆိုတာ မဟုတ်ပါဘူး။ အရေးကြီးတဲ့ ကွန်ပါစ်က စစ်မှန်တဲ့ မြောက်ဘက်ကို ညွှန်တာ မဟုတ်ပါဘူး - ဒါက ကမ္ဘာမှာ သင်အလိုဆုံးအရာကို ညွှန်တာပါ။

Captain တချို့အတွက်၊ ဒါက treasure ကို ညွှန်တယ် - နောက်တစ်ခု ကြီးမြတ်တဲ့ promotion၊ ပြီးပြည့်စုံတဲ့ architecture၊ ပြီးပြည့်စုံတဲ့ codebase။ အခြားသူတွေအတွက်၊ ဒါက လူတစ်ယောက်ကို ညွှန်တယ် - သူတို့ကို စိတ်လှုပ်ရှားစေတဲ့ mentor တစ်ယောက်၊ သို့မဟုတ် ကြီးထွားအောင် ကူညီဖို့ ကြိုးစားနေတဲ့ team member တစ်ယောက်။ နောက်ထပ် အခြားသူတွေအတွက်၊ ဒါက နေရာတစ်နေရာကို ညွှန်တယ် - အားလုံး အလုပ်လုပ်တဲ့ ဒဏ္ဍာရီလို "ပြီးပြည့်စုံတဲ့ project"။

အရာက သင့်ရဲ့ ကွန်ပါစ်က ပြောင်းလဲတယ်။ သင်က ပထမဆုံး sail စတဲ့အခါ လိုချင်ခဲ့တဲ့အရာက အခု သင်လိုချင်တဲ့အရာ မဟုတ်ပါဘူး။ ဒါက မကောင်းတဲ့အရာ မဟုတ်ပါဘူး - ဒါက သင်က ကြီးထွားနေ၊ သင်ယူနေ၊ သင်ဖြစ်သင့်တဲ့ captain ဖြစ်လာနေတယ်ဆိုတာ ဆိုလိုတာပါ။

ဒါပေမယ့် Jack က ဘယ်သူ့ကိုမှ မပြောတဲ့ လျှို့ဝှက်ချက်က - ကွန်ပါစ်က သင်တကယ် လိုချင်တဲ့အရာကို သင့်ကိုယ်သင် ရိုးသားတဲ့အခါမှသာ အလုပ်လုပ်တယ်။ သင်က သင်လိုချင်တယ်လို့ ထင်တဲ့အရာ၊ သို့မဟုတ် အခြားသူတွေက သင်လိုချင်တယ်လို့ မျှော်လင့်တဲ့အရာနဲ့ ဒါကို လှည့်စားလို့ မရပါဘူး။ ဒါက သင့်ရဲ့ နှလုံးသား တကယ် လိုချင်တဲ့အရာကိုသာ ညွှန်တယ်။

ဒါကြောင့် သင့်ကိုယ်သင် မေးကြည့်ပါ၊ Software Captain - သင့်ရဲ့ ကွန်ပါစ်က ဘာကို ညွှန်နေလဲ? ဒါက မှန်တဲ့အရာကို ညွှန်နေလား? သင်က သင်တကယ် လိုချင်တဲ့အရာကို sail လုပ်နေလား၊ သို့မဟုတ် သင်က အခြားသူတွေရဲ့ မျှော်လင့်ချက်တွေရဲ့ လေတွေကိုသာ လိုက်နေလား?

ဘာလို့လဲဆိုတော့ captain တစ်ယောက် ဖြစ်တဲ့အရာက - သင်က သင့်ရဲ့ destination ကို ရွေးချယ်နိုင်တယ်။ သင်က ဘယ် treasure က လိုက်စားဖို့ တန်တယ်ဆိုတာ၊ ဘယ် port က sail လုပ်ဖို့ တန်တယ်ဆိုတာ၊ ဘယ် adventure က အန္တရာယ်ယူဖို့ တန်တယ်ဆိုတာ ဆုံးဖြတ်နိုင်တယ်။ 🗺️

ရေပြင်တွေက အမြဲ ကြမ်းတမ်းမှာ ရှိနေမှာပါ။ ပင်လယ်ဓားပြတွေက အမြဲ ရှိနေမှာပါ။ မုန်တိုင်းတွေက အမြဲ လာမှာပါ။ ဒါပေမယ့် သင်က သင့်ရဲ့ ကွန်ပါစ် တကယ် ဘာကို ညွှန်နေတယ်ဆိုတာ၊ သင်က ဒီကမ္ဘာမှာ သင်အလိုဆုံးအရာကို ရိုးသားစွာ သိရင်၊ သင်က အမြဲ ဘယ်ဘက်ကို sail လုပ်ရမယ်ဆိုတာ သိမှာပါ။

ပြီးတော့ ဒါက ဒီမှာ စစ်မှန်တဲ့ သင်ခန်းစာဖြစ်နိုင်တယ် - ပြီးပြည့်စုံတဲ့ Software Captain တစ်ယောက် ဘယ်လို ဖြစ်ရမယ်ဆိုတာ မဟုတ်ဘဲ၊ သူတို့ ဘယ်ကို သွားနေတယ်ဆိုတာနဲ့ ဘာလို့ သွားနေတယ်ဆိုတာ သိတဲ့ captain ဘယ်လို ဖြစ်ရမယ်ဆိုတာပါ။

ဒါကြောင့် သင့်ရဲ့ ကွန်ပါစ်ကို သတ်မှတ်ပါ၊ Software Captain။ သင်တကယ် လိုချင်တဲ့အရာကို ညွှန်ပါ။ ပြီးတော့ သင်နိုင်သမျှ သတ္တိ၊ ဉာဏ်ပညာ၊ စိတ်ဓာတ်နဲ့ အတူ sail လုပ်ပါ။

Digital seas တွေက စောင့်နေတယ်။ သင့်ရဲ့ crew က အဆင်သင့် ဖြစ်နေတယ်။ သင့်ရဲ့ သင်္ဘောက ကောင်းမွန်တယ်။
